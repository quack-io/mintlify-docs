Tools are the actions your agent can perform. Each tool is a configured integration — an API call, a knowledge base query, an MCP server, or a structured data lookup. Rules control when tools fire; the LLM decides how to use them when no rule applies.

## Tool types

| Type | Description |
| --- | --- |
| `API` | Call any REST endpoint. Supports URL parameter placeholders, auth, and response mapping. |
| `RAG` | Query a knowledge base for retrieval-augmented generation. |
| `MCP` | Connect to a Model Context Protocol server for dynamic tool discovery. |
| `INTERNAL_CONTEXT` | Inject static or dynamic context into the agent's prompt. |
| `STRUCTURED` | Return structured data (forms, cards, carousels) to the user. |
| `GEN_AI` | Run a dedicated LLM sub-task with its own prompt and constraints. |

## Add a tool

<ParamField body="name" type="string" required>
  Display name for the tool.
</ParamField>

<ParamField body="type" type="string" required>
  Integration type. One of `API`, `RAG`, `MCP`, `INTERNAL_CONTEXT`, `STRUCTURED`, or `GEN_AI`.
</ParamField>

<ParamField body="url" type="string">
  Endpoint URL. Supports `{{param}}` placeholders that resolve to agent parameter values.
</ParamField>

<ParamField body="method" type="string">
  HTTP method. One of `GET`, `POST`, `PUT`, `PATCH`, or `DELETE`.
</ParamField>

<ParamField body="authentication" type="string">
  Auth strategy. One of `BEARER_TOKEN`, `API_KEY`, `BASIC`, `OAUTH2`, `HMAC_SHA256`, or `NONE`.
</ParamField>

<ParamField body="request_schema" type="object[]">
  Fields sent in the request body or query string.
  <Expandable title="request_schema fields">
    <ParamField body="key" type="string" required>
      Field name in the request.
    </ParamField>
    <ParamField body="type" type="string" required>
      Data type of the field.
    </ParamField>
    <ParamField body="required" type="boolean">
      Whether this field is required.
    </ParamField>
    <ParamField body="description" type="string">
      Description of the field for documentation and LLM context.
    </ParamField>
    <ParamField body="value_source" type="string">
      Where the value comes from — a parameter code, a literal value, or `auto`.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="response_field_mappings" type="object[]">
  Map response fields to agent parameters.
  <Expandable title="response_field_mappings fields">
    <ParamField body="response_field" type="string" required>
      JSON path in the API response (e.g., `data.order.status`).
    </ParamField>
    <ParamField body="parameter_code" type="string" required>
      Agent parameter code to store the value in.
    </ParamField>
  </Expandable>
</ParamField>

### API tool example

<CodeGroup>

```typescript Node.js
const tool = await aui.agents.tools.create("order-support", {
  name: "Lookup Order",
  type: "API",
  url: "https://api.yourstore.com/orders/{{order-id}}",
  method: "GET",
  authentication: "BEARER_TOKEN",
  request_schema: [
    {
      key: "include_items",
      type: "boolean",
      required: false,
      description: "Whether to include line items",
      value_source: "true",
    },
  ],
  response_field_mappings: [
    { response_field: "order.status", parameter_code: "order-status" },
    { response_field: "order.tracking_url", parameter_code: "tracking-url" },
  ],
});
```

```python Python
tool = aui.agents.tools.create(
```

## Rules

Rules give your agent deterministic behavior. Each rule has a **when** condition and a **then** action. When the condition matches, the action fires — no LLM involvement.

### Add a rule

<ParamField body="description" type="string" required>
  Human-readable description of what this rule does.
</ParamField>

<ParamField body="when" type="object" required>
  Condition group that triggers the rule.
  <Expandable title="when condition group">
    Condition groups combine conditions using logical operators. You can nest groups for complex logic.

    <ParamField body="all" type="object[]">
      All conditions must be true (AND).
    </ParamField>
    <ParamField body="any" type="object[]">
      At least one condition must be true (OR).
    </ParamField>
    <ParamField body="not" type="object[]">
      No conditions may be true (NOT).
    </ParamField>

    Each condition object:
    - `parameter` — the parameter code to evaluate
    - `operator` — comparison operator (`equals`, `not_equals`, `contains`, `is_not_empty`, `greater_than`, `less_than`, etc.)
    - `value` — the value to compare against (when applicable)
  </Expandable>
</ParamField>

<ParamField body="then" type="object" required>
  Action to execute when conditions match.
  <Expandable title="then action">
    <ParamField body="action" type="string" required>
      Action type. One of:
      - `TEXT` — Return a fixed text response
      - `ASK_A_QUESTION` — Ask the user a clarifying question
      - `API` — Call an API endpoint
      - `BLOCK` — Block the agent from responding
      - `ACTIVATE_ANOTHER_TOOL` — Trigger a configured tool
    </ParamField>
    <ParamField body="text" type="string">
      Response text. Used with `TEXT` and `ASK_A_QUESTION`.
    </ParamField>
    <ParamField body="tool" type="string">
      Tool name to activate. Used with `ACTIVATE_ANOTHER_TOOL`.
    </ParamField>
  </Expandable>
</ParamField>

### Rule that triggers a tool

When the customer provides an order ID, automatically look it up.

<CodeGroup>

```typescript Node.js
const rule = await aui.agents.rules.create("order-support", {
  description: "Auto-lookup order when order ID is provided",
  when: {
    all: [
      { parameter: "order-id", operator: "is_not_empty" },
    ],
  },
  then: {
    action: "ACTIVATE_ANOTHER_TOOL",
    tool: "Lookup Order",
  },
});
```

```python Python
rule = aui.agents.rules.create(
    "order-support",
    description="Auto-lookup order when order ID is provided",
    when={
        "all": [
            {"parameter": "order-id", "operator": "is_not_empty"},
        ],
    },
    then={
        "action": "ACTIVATE_ANOTHER_TOOL",
        "tool": "Lookup Order",
    },
)
```

```bash cURL
curl -X POST https://api.aui.io/v1/agents/order-support/rules \
  -H "Authorization: Bearer aui_sk_live_..." \
  -H "x-account-id: acc_01H..." \
  -H "x-organization-id: org_01H..." \
  -H "Content-Type: application/json" \
  -d '{
```

### Rule that blocks a response

Prevent the agent from discussing competitor products.

```bash cURL
curl -X POST https://api.aui.io/v1/agents/order-support/rules \
  -H "Authorization: Bearer aui_sk_live_..." \
  -H "x-account-id: acc_01H..." \
  -H "x-organization-id: org_01H..." \
  -H "Content-Type: application/json" \
  -d '{
    "description": "Block responses about competitor products",
    "when": {
      "any": [
        { "parameter": "user-intent", "operator": "equals", "value": "competitor_comparison" }
      ]
    },
    "then": {
      "action": "TEXT",
      "text": "I can only help with our products. Is there anything else I can assist with?"
    }
  }'
```

### Rule with complex conditions

Escalate when a high-value customer has a shipping delay.

```bash cURL
curl -X POST https://api.aui.io/v1/agents/order-support/rules \
  -H "Authorization: Bearer aui_sk_live_..." \
  -H "x-account-id: acc_01H..." \
  -H "x-organization-id: org_01H..." \
  -H "Content-Type: application/json" \
  -d '{
    "description": "Escalate shipping delays for VIP customers",
    "when": {
      "all": [
        { "parameter": "is-vip", "operator": "equals", "value": "true" },
        { "parameter": "order-status", "operator": "equals", "value": "delayed" }
      ]
    },
    "then": {
      "action": "TEXT",
      "text": "I see your order has been delayed. Let me connect you with our priority support team right away."
    }
  }'
```

## URL parameter placeholders

Tool URLs support `{{param}}` placeholders that resolve to agent parameter values at runtime.

```
https://api.yourstore.com/orders/{{order-id}}/items?status={{item-status}}
```

When the agent calls this tool, `{{order-id}}` is replaced with the current value of the `order-id` parameter and `{{item-status}}` with the `item-status` parameter.

<Note>
  Parameters referenced in URL placeholders must be defined on the agent and have a value at the time the tool is invoked. If a parameter is empty, the tool call will fail.
</Note>

## Authentication strategies

| Strategy | Description |
| --- | --- |
| `BEARER_TOKEN` | Sends `Authorization: Bearer <token>` header. |
| `API_KEY` | Sends the key as a header or query parameter. |
| `BASIC` | HTTP Basic authentication with username and password. |
| `OAUTH2` | OAuth 2.0 flow with automatic token refresh. |
| `HMAC_SHA256` | Signs requests with HMAC-SHA256 for webhook-style auth. |
| `NONE` | No authentication. Use for public APIs. |

<Info>
  Authentication credentials are stored encrypted and never exposed in API responses or logs.
</Info>
